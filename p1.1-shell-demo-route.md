# Crypto Crush | P1.1 | Shell Demo Route

## 1. Shell demo route binding

- demo route is bound to one launch page only
- launch page: `/` (root)
- launch page mounts the shell entry component
- no alternate launch pages
- no direct-link bypass of the shell entry

## 2. Route path

- `/` → splash (shell demo entry)
- `/login` → login
- `/create-account` → create account
- `/home` → home
- `/level-select` → level select
- `/chat` → chat (blocked for guest)
- `/leaderboard` → leaderboard (blocked for guest)
- `/settings` → settings

## 3. Required launch path

`splash → login/home → level_select`

- splash loads first at `/`
- splash routes to `login` if no session, or `home` if signed-in session exists
- guest continue on splash routes directly to `home`
- home routes to `level_select` via primary play action
- path verified end to end

## 4. Cloudflare Pages settings

- project type: Cloudflare Pages
- framework preset: none (static)
- repository: `blackholecapital/crypto-crush-p1`
- production branch: `main`
- preview branches: enabled for `claude/*`
- root directory: `apps/web-public`
- build command: `npm run build`
- build output directory: `dist`
- node version: `20`
- environment variables: none required for shell demo
- SPA fallback: `/index.html` for all unknown routes
- functions directory: none (shell demo has no Pages Functions)
