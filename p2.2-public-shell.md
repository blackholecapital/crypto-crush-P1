## 3. Launch path wiring

- `/` loads `splash`
- splash primary action enters `level_select`
- `level_select → game` (via approved WA entry payload)
- `game → result` (via approved WA exit payload)
- `result → retry` (fail + `nextRoute = "retry"`)
- `result → next_level` (win + `nextRoute = "next_level"`)
- `result → level_select` (back_out + `nextRoute = "level_select"`, or direct return button)

## 4. Shell-owned state

Owned:

- `currentScreen`
- `selectedLevel`
- unlocked level memory (browser session only)
- guest identity display: `username = "Guest"`, `sessionId = null`
- local settings constants: `audioEnabled`, `hapticsEnabled`

Not owned:

- board
- score calculation
- moves logic
- goal logic
- cascade logic

## 5. WA contract consumption

Entry payload built (not redefined):

- `levelId`, `username`, `sessionId`, `entrySource`, `audioEnabled`, `hapticsEnabled`

Exit payload accepted (not redefined):

- `levelId`, `username`, `outcome`, `score`, `movesUsed`, `movesRemaining`, `nextRoute`

Implemented in `apps/web-public/public/shell/contracts.js` via `buildEntryPayload()` and `isValidExitPayload()`.

## 6. Level unlock wiring

- level 1 unlocked by default
- on win, unlock next level up to 5
- on fail, no new unlock
- on back_out, return to `level_select`
- unlock memory preserved for current browser session only (in-memory, not persisted)

## 7. Result routing

- `win` + `nextRoute = "next_level"` → launch next unlocked level
- `fail` + `nextRoute = "retry"` → relaunch same level
- `back_out` + `nextRoute = "level_select"` → return to level select
- direct return to `level_select` always available via secondary button on result screen
